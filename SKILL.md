# 求职招呼生成助手

## 能力

根据你给的岗位 JD（Job Description）和你本地的简历，自动生成精简、有针对性、每句话都有信息量的求职招呼语或面试自我介绍。

## 核心规则（写之前必须遵守）

### 话术三原则
1. **一句话一个点**——不说"我熟练 Python"，说"用 Python 写过批量测试脚本，自动跑 200+ 用例输出报告"
2. **不说虚词**——不用"熟练掌握"、"具备一定的"、"我了解"这类空话，直接从技能/动作开始写
3. **不硬凑**——JD 要求的经历如果没有，不编不绕，换你简历里最相关的去打

### 匹配策略
- 区分 JD 中的 **硬性门槛**（如学历、学位证、专业）→ 不满足直接提醒用户
- 区分 **核心技术要求**（如 Python、示波器、Serdes 测试）→ 重点匹配
- 区分 **软性加分**（如学习能力强、团队协作）→ 可以在最后提一句
- 对每一项匹配标注：强匹配 ✅ / 弱匹配 ⚠️（可类比或方向相关）/ 缺失 ❌

## 工作流程

### Step 1: 获取输入
确认两样东西：
1. **JD（岗位描述）**——用户粘贴
2. **简历路径**——询问用户用哪份简历（列出已知的简历文件让用户选，或让用户给路径）

### Step 2: 解析 JD
提取出以下结构：
```
硬性门槛: [本科及以上, 学位证, ...]
技术关键词: [Python, C, 示波器, Serdes, ...]
工作内容: [芯片测试, 批量数据处理, 测试开发...]
加分项: [有硬件测试经验, 熟悉C语言...]
```

### Step 3: 匹配简历
对照简历逐项匹配，输出匹配度清单给用户看：
```
✅ Python → 简历有：写过自动化测试脚本
✅ 示波器/万用表 → 简历有：实习期间使用
⚠️ Serdes/光通信 → 简历无直接经验，但 STM32 通信调试经验可类比表述
❌ 某项要求 → 简历无，提示用户
```

### Step 4: 生成招呼语
生成 2 个版本供用户选择：

| 版本 | 长度 | 用途 |
|------|------|------|
| 打招呼版 | 80-120 字 | 投简历时附带的第一段话 |
| 面试版 | 150-250 字 | 1 分钟口头自我介绍 |

**每一句话必须对应一个 JD 要求或差异化优势，不允许有无信息量的句子。**

### Step 5: 精简检查（生成后自动执行）
检查生成的文本，去掉明显的废话：
- 删除"熟练掌握""具备一定"这类空洞修饰词
- 删除并列堆砌技能（"熟悉 Python、C、Java"改为最匹配的那个+场景）
- 删除重复表达

### Step 6: 自然度润色 —— 去 AI 味（生成后自动执行）
把上一步的文本改成"人话"，核心规则：

**① 加生活感和语气**
- 不直接甩技能，加个"我"或"之前/之前实习/在校期间"开头
- 适当加"了"、"过"、"的"、"其实"、"平时"这类口语助词
- 不要每句都像简历 bullet point，要有自然的句子衔接

**② 变堆砌为叙述**
- ❌ "熟练 Python，实习期间写过自动化测试脚本批量验证模块通信。"
- ✅ "之前在实习的时候用 Python 写过自动化测试脚本，批量跑完还能自动出报告。"

**③ 信息密度降一档**
- 不用每句都带数据和指标，适当留白
- 10 个字能说清的事，不用硬塞到 8 个字

**④ 读一遍检查**
- 大声读一遍，感觉像不像人当面说的
- 不像就改到像为止

**润色前后对比示例：**

```
精简后（AI味重）：
"熟练 Python，实习期间写过自动化测试脚本批量验证模块通信。STM32项目独立完成传感器精度测试和MQTT联调，已拿双证，随时到岗。"

润色后（像人话）：
"之前实习的时候用 Python 写过自动化测试脚本，批量跑完还能自动出报告。做项目的时候也用 STM32 做过传感器测试和 MQTT 上云联调，软硬件都接触过。已经拿到毕业证和学位证了，随时能到岗。"
```

### Step 7: 用户确认
让用户选择版本或提出修改意见（太长/太短/多突出XX/换个语气），直接迭代，不需要重新贴 JD 和简历。

## 触发方式

当用户提到以下内容时，主动建议使用本 Skill，并询问是否要运行 `/resume-pitch-crafter`：
- "帮我写个招呼语" / "写个打招呼"
- "根据 JD 写自我介绍" / "自我介绍"
- "我要投简历" / "投递这个岗位"
- "帮我匹配一下这个岗位" / "看看我合不合适"
- "生成一段求职开场白"

## 用法

敲命令主动使用：

```
/resume-pitch-crafter
```

或者一步到位：

```
/resume-pitch-crafter + paste JD内容
```

## 边界与约束

- ❌ 不可以编造简历中没有的经历
- ✅ 可以对已有经历换角度表述（同一项目，AI岗突出LLM，测试岗突出测试流程）
- ✅ 可以建议用户补充缺失的关键技能
- ✅ 生成后必须说明"这段话里每个点对应你简历的哪里"，让用户心里有数
